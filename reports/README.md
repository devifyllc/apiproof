# Reports Directory

This directory stores generated HTML validation reports.

## Directory Structure

```
reports/
├── soap/     # SOAP validation reports
└── rest/     # REST validation reports
```

## Generated Reports

### Individual Validation Reports
- **SOAP**: `{ResponseType}-validation-report.html`
- **REST**: `{endpoint}-validation-report.html`
- **Contains**: Detailed validation results, violations, and metadata

### Summary Reports
- **SOAP**: `validation-summary.html`
- **REST**: `validation-summary.html`
- **Contains**: Aggregated results from batch validation runs

## Report Features

- ✅ **Pass/Fail Status**: Clear indication of validation outcome
- 📊 **Violation Details**: Specific contract violations with descriptions
- 📝 **Metadata**: Timestamps, file paths, system version
- 🎨 **Beautiful UI**: Clean, professional HTML design
- 📱 **Responsive**: Works on desktop and mobile browsers

## Notes

- Reports are automatically generated by validation tools
- These files are gitignored by default (see `.gitignore`)
- Open reports in any web browser
- Reports are self-contained HTML files (no external dependencies)
